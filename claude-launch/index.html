Web environment ready

Claude Code on the web, not just on my laptop.

Sessions launch in cloud containers from any device. The repo gets cloned fresh each time, the work goes back through a PR. This means:

  • Phone, tablet, any browser. I can kick off a task from a coffee shop and review the PR on the way home.
  • Ephemeral isolation. Each session is its own sandbox. Nothing crosswires with the laptop.
  • Scoped repo access. The web env knows exactly which repos this branch can touch. No accidental writes elsewhere.
  • The 8 GB Chromebook is no longer the bottleneck. Heavy lifting moves to the cloud, the laptop stays cool.

The bar I hold

No slop ships.

The team's reputation is the moat. The rules I encoded keep the bar high even when agents are doing the typing:

  • Live integration evidence. Autonomous-green is necessary, never sufficient. A real request flows through the real architecture before "done."
  • Codex adversarial review. A second model tries to refute the work before merge. Catches what a single perspective misses.
  • No AI tells. Public-facing copy passes the de-slop check. No em dashes, no "as an AI", no cliche openers.
  • Memory discipline. Nothing durable lives in a machine-local cache. If it matters, it's in git, shared with the team.
