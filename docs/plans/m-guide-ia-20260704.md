    var BLUES_KEY = {
      label: 'Blues',
      degrees: [ { roman: 'I7', semis: 0 }, { roman: 'IV7', semis: 5 }, { roman: 'V7', semis: 7 } ],
      soloDefault: 'blues'
    };
    function bluesKey(root) {            // -> [{roman:'I7', chord:'C7', root:'C', quality:'7'}, ...]
      var pc = pcOf(root); if (pc < 0) return [];
      return BLUES_KEY.degrees.map(function (d) {
        var r = spell(pc + d.semis);
        return { roman: d.roman, chord: r + '7', root: r, quality: '7' };
      });
    }
    // chordTones(chord) -> pcs (triad + optional 7th). App vocabulary only:
    // '', m, dim, aug via suffixQuality; then maj7 -> +11, any other trailing 7 -> +10.
    // Unknown suffixes fall back to the bare triad. Pure pc arithmetic; exported for targeting.

## 2. CHORD-TONE TARGETING (Studio)

**Class model** - one class per rendered in-scale dot, precedence root > chord > blue > scale:

| Class | Rule (pure pc arithmetic) |
|---|---|
| root | target active: pc === CHORD root pc; no target: pc === scale root (current behavior) |
| chord | pc in Circle.chordTones(target) minus root |
| blue | blues scale's b5 (scaleRootPc+6) - whenever the blues scale renders, targeted or not |
| scale | remaining in-scale tones |
| rub (modifier) | target dominant-quality (chordTones has root+4 AND root+10) -> rubPc = (chordRootPc+3)%12 IF in scale. Marks Eb over C7, Bb over G7 in C blues; nothing over F7. Dashed ring |

**D-TARGET (intersection-only):** only chord tones INSIDE the current scale light up. No ghost dots for out-of-scale chord tones (E over C blues) - the blues card carries that advice; ghost dots = named deferral.

**Interaction - one target surface: chords-in-key cells.** Tap = play (existing) + toggle target; cell gets .targeted outline; bridge t.seq is NOT a second source. Static caption under fretboard while active (A9, interpolates rendered labels): "Showing C7 inside A blues - accent = chord root, filled = chord tones." Plain div, not a Notable.

**API - extend, don't overlay; preserve position:** Diagram.scale(opts) gains `opts.tones = { byPc: {pc: class}, rubPc }`; per dot `cls = tones.byPc[pc] || (isRoot ? 'root' : 'scale')` -> class kxDot kx-<cls> + CSS-var inline styles (--kx-chord, --kx-blue in tracks.css, theme-safe); rubPc adds kx-rub + stroke-dasharray "3 2". **tones absent -> byte-identical current render (regression-locked in diagram.dom.test.js).** KeyExplorer.renderScale passes opts.tones; boxWrap gains setTones(tones) re-rendering diagBox only, startFret preserved. tracks.js pure exported `targetTones(scalePcs, scaleRootPc, chordName)` -> `{byPc, rubPc}|null`; `defaultTones(bundle)` -> blue map for blues else null. Chip select() re-applies active target against new bundle.

**Determinism (binding):** every class = pc set intersection on Circle pcs + parsed suffix. No model calls, no per-chord curated data, no heuristics. Cards are the only curated text (A9 static).
