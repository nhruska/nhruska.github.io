Theory audit - detail (phase 2a)

0 BUGS
1008 checks: every pitch class, triad quality, and roman numeral is theory-correct across 12 roots x 4 modes (Major, Minor, Mixolydian, Dorian).
128 spelling deltas
In 24/48 contexts the canonical-sharp display differs from written-music convention (all pitch-correct; zero are errors of sound).

Methodology (attackable, by design)

Ground truth built independently of the app: letter-sequential spelling (7 letters, each once; accidentals bb..## absorb differences) + triad qualities from stacked thirds + conventional key names by fewest-accidentals (tie -> sharps). App outputs via its own exported functions (Circle.spellScale, Songbook.chordsFromDegrees, Songbook.romanInKey). Every delta classified BUG (wrong sound/label) vs CONVENTION (right sound, non-traditional spelling). Weakness acknowledged: the same author wrote both sides - hence the professor pass (D4).

Roman-numeral convention - what the app actually does (the professor's V1/V2 questions)

CaseApp showsConvention family
Diatonic chord, in-mode (D in E Mixolydian)VII (plain mode-degree)Classical/modal-analysis style
Same chord, borrowed (D in E Major)bVII (chromatic, major-relative)Pop/lead-sheet style
Minor key degrees 3/6/7 (C Minor)III VI VIIClassical (lead-sheet would write bIII bVI bVII)
Diminishedlowercase + ° (vii°)Both
Comment: the hybrid is internally consistent and defensible, but it means the SAME chord relabels when the mode toggles (bVII <-> VII by design). P2 likely reads the modal plain-VII as classical-correct; P4's jam vocabulary ("the Mixolydian bVII") may expect the flat.
